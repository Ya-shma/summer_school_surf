# Тест-кейсы для веб-приложения «Скалодром»

## Фича 1: Просмотр расписания с фильтрами

### TC-1.1: Отображение списка слотов
**Предусловия:** Клиент авторизован.  
**Шаги:**
1. Открыть http://localhost:5173
2. Перейти на вкладку «Расписание»

**Ожидаемый результат:**
- Отображается список слотов на ближайшие 7 дней
- Каждый слот содержит: дата/время, формат, инструктор, свободные места, цена
- Слоты отсортированы по времени старта (ближайшие выше)

**Статус:** ✅ PASS

---

### TC-1.2: Фильтрация по формату
**Предусловия:** Клиент авторизован, есть слоты разных форматов.  
**Шаги:**
1. Нажать кнопку «⚙️ Фильтры»
2. Выбрать «🌱 Новички»
3. Нажать «Применить»

**Ожидаемый результат:**
- В списке отображаются только слоты формата «beginner»
- Сверху появляется чип «Новички ✕»
- В DevTools Network: `GET /api/slots/?format=beginner`

**Статус:** ✅ PASS

---

### TC-1.3: Сброс фильтра через чип
**Предусловия:** Активен фильтр по формату.  
**Шаги:**
1. Кликнуть на чип «Новички ✕»

**Ожидаемый результат:**
- Список обновляется, показываются все слоты
- Чип исчезает
- В Network: `GET /api/slots/` (без параметров)

**Статус:** ✅ PASS

---

### TC-1.4: Фильтр «Только свободные»
**Предусловия:** Есть заполненные слоты.  
**Шаги:**
1. Открыть фильтры
2. Включить «Только свободные»
3. Нажать «Применить»

**Ожидаемый результат:**
- Заполненные слоты скрыты
- Появляется чип «Только свободные ✕»

**Статус:** ✅ PASS

---

### TC-1.5: Empty state
**Предусловия:** Нет слотов под условия фильтра.  
**Шаги:**
1. Применить фильтр, по которому нет слотов

**Ожидаемый результат:**
- Отображается «Пока нет доступных тренировок»
- Кнопка «Сбросить фильтры» активна

**Статус:** ✅ PASS

---

## Фича 2: Запись на тренировку

### TC-2.1: Запись со своими скальниками
**Предусловия:** Клиент авторизован, есть свободный слот.  
**Шаги:**
1. Выбрать свободный слот
2. Нажать «Записаться · 1500 ₽»
3. В модалке выбрать «🧗 Свои скальники»
4. Нажать «Записаться · 1500 ₽»

**Ожидаемый результат:**
- Появляется тост «Запись создана!»
- Переход в «Мои бронирования»
- В БД: `bookings` со статусом `booked`, `equipment_type='own'`
- `slots.booked_places` увеличился на 1

**Статус:** ✅ PASS

---

### TC-2.2: Запись с прокатом
**Предусловия:** Клиент авторизован, есть свободный слот, `rental_available > 0`.  
**Шаги:**
1. Выбрать слот → «Записаться»
2. Выбрать «🎒 Прокат (+300 ₽)»
3. Нажать «Записаться · 1800 ₽»

**Ожидаемый результат:**
- Бронь создана со статусом `booked`, `equipment_type='rental'`
- `slots.rental_available` уменьшился на 1
- Итоговая цена: 1800 ₽

**Статус:** ✅ PASS

---

### TC-2.3: Согласие с ТБ при первой записи
**Предусловия:** Клиент авторизован, `safety_rules_accepted=false`.  
**Шаги:**
1. Нажать «Записаться»
2. В модалке поставить галочку «Ознакомлен с правилами»
3. Нажать «Принять и продолжить»

**Ожидаемый результат:**
- POST /api/auth/profile → 200
- Автоматически открывается модалка записи
- Флаг `safety_rules_accepted=true` в профиле

**Статус:** ✅ PASS

---

### TC-2.4: Запись ребёнка
**Предусловия:** Клиент авторизован.  
**Шаги:**
1. В модалке записи включить «Записать ребёнка»
2. Ввести возраст: 8
3. Нажать «Записаться»

**Ожидаемый результат:**
- Бронь создана с `child_age=8`
- В «Мои бронирования» отображается «Ребёнок, 8 лет»

**Статус:** ✅ PASS

---

### TC-2.5: Двойная бронь
**Предусловия:** Клиент уже записан на слот.  
**Шаги:**
1. Попытаться записаться на тот же слот

**Ожидаемый результат:**
- Ошибка «Вы уже записаны на эту тренировку» (409 `double_booking`)

**Статус:** ✅ PASS

---

### TC-2.6: Гейткинг для новичка
**Предусловия:** Клиент с `is_allowed_to_rope=false`.  
**Шаги:**
1. Выбрать слот формата «Опытные»
2. Нажать «Записаться»

**Ожидаемый результат:**
- Ошибка «Сначала пройдите вводную тренировку для новичков» (403 `rope_access_required`)

**Статус:** ✅ PASS

---

### TC-2.7: Блокировка клиента
**Предусловия:** Клиент с `blocked_until > now()`.  
**Шаги:**
1. Нажать «Записаться»

**Ожидаемый результат:**
- Ошибка «Вы заблокированы до N» (403 `client_blocked`)

**Статус:** ✅ PASS

---

### TC-2.8: Idempotency-Key в запросах
**Предусловия:** Клиент авторизован.  
**Шаги:**
1. Записаться на тренировку
2. Открыть DevTools → Network → POST /api/bookings/

**Ожидаемый результат:**
- В заголовках запроса есть `Idempotency-Key: xxxxxxxx-xxxx-...`

**Статус:** ✅ PASS

## Фича 3: Отмена записи

### TC-3.1: Ранняя отмена (≥ 6 часов)
**Предусловия:** Клиент записан на тренировку через 2 дня.  
**Шаги:**
1. Открыть «Мои бронирования»
2. Нажать «Отменить»
3. Подтвердить в модалке

**Ожидаемый результат:**
- Бронь переходит в статус `cancelled_by_client`
- `slots.booked_places` уменьшился на 1
- Если был прокат → `slots.rental_available` увеличился на 1

**Статус:** ✅ PASS

---

### TC-3.2: Поздняя отмена (< 6 часов)
**Предусловия:** Клиент записан на тренировку через 3 часа.  
**Шаги:**
1. Открыть «Мои бронирования»

**Ожидаемый результат:**
- Кнопка «Отменить» отсутствует
- Пояснение «Отмена возможна не позднее чем за 6 часов»
- Кнопка «💬 Написать Оле» → открывает `https://t.me/vertical_gym`

**Статус:** ✅ PASS

---

### TC-3.3: Отмена через детали брони
**Предусловия:** Клиент записан на тренировку через 2 дня.  
**Шаги:**
1. Тапнуть по карточке брони
2. В модалке деталей нажать «Отменить запись»
3. Подтвердить

**Ожидаемый результат:**
- Отмена успешна, возврат в список

**Статус:** ✅ PASS

---

### TC-3.4: Отмена скалодромом
**Предусловия:** В БД слот со статусом `cancelled`, `cancel_reason='Профилактика'`.  
**Шаги:**
1. Открыть «Мои бронирования»

**Ожидаемый результат:**
- Баннер «🚫 1 тренировок отменено скалодромом»
- Карточка со статусом «🚫 Отменено скалодромом»
- Причина «Профилактика» отображается
- Кнопка «Выбрать другое время» → открывает расписание с фильтром

**Статус:** ✅ PASS

---

### TC-3.5: Нарушения и блокировка
**Предусловия:** Клиент сделал 3 поздних отмены.  
**Шаги:**
1. Проверить в БД: `violations_count=0`, `blocked_until=now+7 дней`
2. Попытаться записаться

**Ожидаемый результат:**
- Ошибка «Вы заблокированы до N»

**Статус:** ✅ PASS
