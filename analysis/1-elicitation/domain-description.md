# Описание домена

## Бизнес

«Вертикаль» — небольшой скалодром в бывшем складском ангаре, работающий третий сезон. Клуб проводит групповые тренировки по скалолазанию длительностью ~1.5 часа в одном из двух форматов: **новичковый** (болдеринг с инструктажем) и **опытный** (трассы с верёвкой). Каждую тренировку ведёт инструктор (сейчас их пятеро, в сезон нанимаются подработчики).

## Процесс

Владелец заранее формирует расписание на неделю вперёд: для каждой тренировки задаются дата и время старта, формат, назначенный инструктор и число мест. Клуб располагает 20 комплектами прокатного снаряжения (скальники + страховочная система); для новичковых групп берут не более 8 человек (тяжелее уследить по технике безопасности), для опытных — до 16. Клиент видит свободные слоты тренировок и записывается, указывая, придёт он со своими скальниками или возьмёт снаряжение напрокат. До тренировки клиент может отменить запись — при ранней отмене место освобождается, поздняя отмена оставляет место простаивать. Сейчас запись клиентов ведётся вручную (Telegram + тетрадь), что приводит к двойным записям и путанице, особенно в часы пик.

## Цель клиентского приложения

Дать клиентам самостоятельную онлайн-запись на тренировки (просмотр и фильтрация слотов, бронирование, выбор снаряжения, отмена, свои бронирования, оценка инструктора), исключив ручную путаницу и двойные брони. Управление расписанием и работу инструкторов/владельца обеспечивает существующая инфраструктура; приложение потребляет её данные через API.

## Ключевые объекты домена

- **Тренировка / слот** — конкретное занятие группы: дата/время старта, формат, инструктор, всего и свободно мест, свободный прокатный фонд. Создаётся в существующей инфраструктуре; клиенту доступен только для чтения.
- **Формат** — вариант тренировки (новичковый / опытный), определяет потолок мест (справочные данные, read-only).
- **Инструктор** — ведёт тренировку; для клиента — справочные данные слота (read-only).
- **Прокатное снаряжение** — инвентарь (всего 20 комплектов); может выдаваться напрокат.
- **Запись (бронь)** — резерв места клиентом на конкретный слот: своя/прокатная экипировка, статус, возраст участника (для детей — указывается при записи).
- **Клиент** — записывается на тренировки, отменяет запись, оценивает инструктора. Имеет флаги «допущен к верёвкам» и «постоянный клиент».
- **Оценка инструктора** — 1–5 звёзд, видна только владельцу в админке.
- **Список ожидания** — очередь клиентов на конкретный заполненный слот.

## Пользователи и роли

Скоуп текущей работы — только клиентское мобильное приложение и API для него. Роли Инструктор и Владелец/Администратор обслуживаются существующей инфраструктурой (бэкенд, расписание, админка) и в этот скоуп не входят.

| Роль | Описание | Цели | В скоупе |
|---|---|---|---|
| Клиент | Гость скалодрома, лазает на скалодроме. Пользуется с телефона. | Увидеть и отфильтровать свободные слоты, записаться (только себя), выбрать свои/прокатные скальники, видеть свои бронирования, отменить запись, оценить инструктора, получать напоминания. | Да |

Инструктор и Владелец/Администратор взаимодействуют с системой через уже реализованные интерфейсы существующей инфраструктуры. Клиентское приложение лишь потребляет подготовленные ими данные (слоты, форматы, инструкторы) через API.

## Ограничения

### Бизнес-ограничения

- Всего 20 комплектов прокатного снаряжения — общий потолок прокатного фонда.
- Новичковая группа — не более 8 человек; опытная — до 16. Лимит жёстко задан по формату, инструктор менять не может.
- **Гейткинг по формату**: клиент без флага «допущен к верёвкам» (выставляется владельцем в админке после вводного инструктажа) не может записаться на опытный слот.
- **Контроль проката**: если клиент выбрал «нужно прокатное снаряжение», а свободного фонда нет — запись запрещена. Клиент со своими скальниками может записаться даже при нулевом фонде.
- **Одна бронь = один клиент** (взрослый или ребёнок). Родитель может записать ребёнка со своего аккаунта, указав при записи возраст ребёнка (например, «запишу ребёнка 8 лет»). У ребёнка нет отдельного аккаунта — он всегда приходит с родителем.
- **Дети**: ходят с 6 лет, только в сопровождении родителя. При записи ребёнка родитель указывает возраст; отдельный аккаунт для ребёнка не требуется.
- **Политика отмен**: бесплатная отмена не позднее чем за 6 часов до старта; кнопка «Отменить» в приложении блокируется ближе этого порога. Поздняя отмена или неявка фиксируется как нарушение; после 3 нарушений подряд — автоматическая блокировка записи на неделю. Денежных штрафов нет (оплата офлайн).
- **Оплата офлайн** (наличные / перевод на карту) — приложение лишь фиксирует запись и показывает цену (1500 ₽ фиксированно для любого формата).
- **Список ожидания**: клиент явно нажимает «уведомить, если появится место» на заполненном слоте; при освобождении места push получает первый в очереди.
- **Программа лояльности в V1**: статус «постоянный клиент» присваивается автоматически после 10 тренировок; привилегии — бейдж в профиле и приоритетная запись (открытие слотов за 2 дня до тренировки вместо 1 дня для остальных).
- **Оценки инструктора**: только звёзды 1–5, без текстовых комментариев; видны только владельцу в админке (клиентам не показываются). Запрос на оценку приходит через 1 час после окончания тренировки; поставить можно только после фактического посещения (факт посещения определяется бэкендом по времени начала слота и наличию активной брони).
- **Период списка слотов по умолчанию — 7 дней (R-027)**. По умолчанию клиенту показываются слоты на ближайшие 7 дней от текущего момента. Более длинный период доступен через фильтр дат. При отсутствии расписания на ближайшие дни показывается empty state («Пока нет доступных тренировок»).
- **Отмена тренировки скалодромом / по профилактике (R-008)**. Если тренировку отменяет скалодром (организатор или из-за профилактики/проверки снаряжения), бронь на стороне клиента не удаляется, а получает статус «Отменена скалодромом» с указанием причины; клиенту приходит push-уведомление с кнопкой «Выбрать другое время» (открывает расписание с фильтром по тому же формату). Повторная запись на отменённый слот запрещена. Само действие отмены слота выполняется в существующей инфраструктуре (вне скоупа клиентского приложения).
- **Уведомления — только push** (за день и за час до тренировки). SMS не используются. Если клиент запретил push на уровне ОС — показывается одноразовое предупреждение «вы не будете получать напоминания», функционал не блокируется.
- **Регистрация — только по номеру телефона с SMS-кодом**. Email и соцсети не поддерживаются. При регистрации обязательны галочки согласия на обработку данных; перед первой записью — дополнительная галочка «ознакомлен с правилами и техникой безопасности».
- **Обратная связь** — кнопка «Написать Оле» в профиле открывает чат в Telegram; встроенного чата в приложении нет.

### Технические / платформенные

- **Mobile-first**: клиентское приложение рассчитано на телефон (использование в зале и дома).
- Клиентское приложение взаимодействует с существующим бэкендом через API; слоты, форматы и инструкторы доступны только для чтения.
- **Граница серверной интеграции (R-004)**. Детали серверной реализации — транзакционность, SLA, маппинг сущностей на внутренние модели бэкенда — вне скоупа клиентского приложения. Существующий бэкенд — black-box источник истины: гарантия «0 двойных броней» обеспечивается на его стороне (атомарная проверка свободных мест при бронировании). Со стороны клиента граница закрывается контрактом API: приложение полагается на ответы бэкенда и корректно обрабатывает отказ, если мест уже нет.
- **Данные существующей инфраструктуры (R-015)**. Это учебный/тестовый проект, исторических данных и легаси-схемы нет. Канонической схемой считается контракт API: бэкенд по условию отдаёт ровно те поля, что описаны в контракте, включая прокатный тариф, адрес скалодрома, свободный прокатный фонд. Поэтому миграция/backfill и стратегия дозаполнения исторических слотов вне скоупа — отдельных требований к ним не формулируем.

## Проект

- **Срок** — к началу следующего сезона, ~2 месяца (жёсткая цель).
- **Бюджет ограничен** → начинаем с самого нужного, «навороты» позже.

### Границы скоупа

Скоуп — клиентское мобильное приложение и API для него. Управление расписанием, роли инструктора и владельца, админка — это существующая инфраструктура, вне скоупа.

| В скоупе клиентского приложения | Вне скоупа (существующая инфраструктура / не делаем сейчас) |
|---|---|
| Регистрация/вход клиента по телефону (SMS) и имени | Роли Инструктора и Владельца/Администратора, админ-экран |
| Просмотр и фильтрация списка слотов (дата, формат, наличие мест, инструктор) | Создание и редактирование расписания (слоты приходят из существующего бэкенда через API) |
| Самостоятельная запись (только себя, без гостей) | Оценка инструктора, видимая клиентам (публичный рейтинг) — не делаем |
| Выбор «свои скальники / прокат», контроль лимита мест и прокатного фонда | Отметка явки/неявки, отмена слота инструктором + массовое уведомление |
| Гейткинг по формату (флаг «допущен к верёвкам») | Онлайн-оплата; ручная отметка «оплачено» |
| Список своих бронирований (активные + история) | Автоматический учёт погоды / прогноза |
| Отмена записи с правилом 6 часов | Расширенные механики лояльности (скидки, бесплатные тренировки) |
| Цена и фиксация записи (оплата офлайн, 1500 ₽) | Перенос записи как единая операция (реализуется как отмена + новая запись) |
| Оценка инструктора (1–5 звёзд, приватно для владельца) | Текстовые комментарии к оценкам |
| Push-напоминания о записи (за день и за час) | SMS/WhatsApp-каналы уведомлений |
| Список ожидания с push-уведомлением при освобождении места | Онбординг с обучающими экранами |
| Программа лояльности в минимальном виде (бейдж + приоритет открытия слотов) | Групповые брони (запись нескольких человек с одного аккаунта) |
| Кнопка «Написать Оле» → переход в Telegram | Встроенный чат поддержки в приложении |
