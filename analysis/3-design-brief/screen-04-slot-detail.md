Экран: Карточка слота (детальная)
Связанный файл: `design-brief.md`
UC: UC-2, UC-3, UC-5
Порядковый номер: 4

Цель
Дать всю информацию для принятия решения и записаться.

Связь с требованиями
FR: FR-9a, FR-10, FR-11, FR-13, FR-14, FR-15, FR-45, FR-51, FR-54, FR-55
US: US-4, US-6, US-7, US-8, US-9, US-10
UC: UC-2, UC-3

Структура экрана
Заголовок: дата, время, формат (бейдж).
Инструктор: имя + фото (если есть).
Статус мест: прогресс-бар `5 / 8 мест` + `12 / 20 комплектов проката свободно`.
Цена: `1500 ₽` (фиксированно) + стоимость проката (если выбран) — FR-45.
Адрес: текстом (без карты — по ответам Оли).
Чек-лист «Что взять»: короткий текст.
Ссылка «Правила и техника безопасности» → внешний URL.
CTA-зона (внизу, зафиксирована):
- Активная: «Записаться · 1500 ₽».
- Неактивная с пояснением (гейткинг / блокировка / мест нет / отменён).

Состояния
| Состояние | Поведение |
|---|---|
| Загрузка | Skeleton + актуальный запрос при каждом открытии (NFR-31) |
| Данные | Полная карточка |
| Ошибка | Баннер «Не удалось загрузить» + Retry |
| Offline | Баннер «Данные могли устареть» |

Заметки для дизайнера
- При каждом открытии карточки — актуальный запрос с бэкенда (NFR-31).
- Цена — read-only из `price_total` ответа API (FR-45).
- CTA-зона зафиксирована внизу экрана (sticky footer) для быстрого доступа на мобильных и десктопе.
- Все состояния CTA должны быть проработаны в макете.
- Если слот заполнен — показывать кнопку «Уведомить, если появится место» (Alert List, FR-53).
- Если клиент не допущен к верёвкам — CTA disabled, текст «сначала пройдите вводную тренировку для новичков» (FR-55).
- Если слот отменён скалодромом — CTA disabled, текст «Отменена скалодромом: [причина]» + ссылка на выбор другого времени (FR-46).
