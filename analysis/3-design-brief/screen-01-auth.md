Экран: Вход / регистрация

Связанный файл: `design-brief.md`
UC: UC-1 (Регистрация и вход по SMS OTP)
Порядковый номер: 1

## Цель

Максимально быстро ввести клиента в сессию.

## Связь с требованиями

- FR: FR-1, FR-2, FR-43, FR-50
- NFR: NFR-3 (лёгкая регистрация), NFR-19 (антифрод OTP)
- US: US-1
- UC: UC-1

## Структура экрана

### Шаг 1. Ввод телефона

- Логотип «Вертикаль» + заголовок «Войти по телефону».
- Поле ввода телефона с маской `+7 (___) ___-__-__`.
- Чекбокс «Согласен на обработку персональных данных» + ссылка на политику.
- CTA «Получить код» (активна только при валидном телефоне + галочке).
- Мелкий текст: «Войдёте по SMS-коду. Это бесплатно для вас».
- Ссылка на политику конфиденциальности — кликабельный текст.

### Шаг 2. Ввод OTP

- Заголовок «Код отправлен на +7 *** *** ** 67».
- 4–6 полей для кода (авто-фокус, WebOTP API для авто-чтения из SMS — ограниченная поддержка в браузерах).
- Таймер «Отправить повторно через 00:30» → становится кнопкой.
- Счётчик неверных попыток: после 3 — экран блокировки на 15 минут с таймером (NFR-19).

### Шаг 3. Заполнение профиля (только для новых)

- Имя (обязательно).
- Возраст (опционально, числовое поле).
- День рождения (опционально, date-picker).
- CTA «Продолжить».

## Состояния экрана

### Шаг 1. Ввод телефона

| Состояние | Поведение |
|---|---|
| Пустое | Поле не заполнено → CTA disabled |
| Отправка кода | Loading-индикатор на кнопке |
| Ошибка отправки SMS (E1 UC-1) | Тост «Не удалось отправить код, попробуйте позже» |
| Rate-limit (E3 UC-1) | Сообщение «Слишком много попыток, попробуйте через N минут» |
| Нет сети (E4 UC-1) | Баннер «Нет соединения» + кнопка «Повторить» |

### Шаг 2. Ввод OTP

| Состояние | Поведение |
|---|---|
| Неверный код (E2 UC-1) | «Неверный код, попробуйте ещё раз», поле активно |
| 3 неверных попытки | Экран блокировки на 15 минут с таймером (NFR-19) |
| Успех | Переход на шаг 3 (для новых) или на «Расписание» (для существующих) |

## Матрица ошибок

| Код | HTTP | UX-реакция |
|---|---|---|
| Недоставка SMS | — | «Не удалось отправить код, попробуйте позже» |
| invalid_code | 400/401 | «Неверный код, попробуйте ещё раз» |
| 429 + retry_after | 429 | «Слишком много попыток, попробуйте через N минут» |
| Нет сети | — | Баннер «Нет соединения» + Retry |

## Заметки для дизайнера

- Телефон маскируется в UI: `+7 *** *** ** 67` (NFR-20).
- WebOTP API для авто-чтения SMS-кода — нативная возможность браузера, предусмотреть (ограниченная поддержка).
- Таймер повторной отправки — 30–60 секунд (NFR-19).
- Блокировка после 3 неверных попыток — 15 минут с видимым таймером.
- Поле «Возраст» и «День рождения» — опциональные, не блокируют CTA.
