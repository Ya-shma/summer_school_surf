Экран: Предупреждение о push

Связанный файл: `design-brief.md`
UC: UC-8 (Получение push-уведомлений)
Порядковый номер: 11

## Цель

Один раз объяснить клиенту, что он не будет получать напоминания.

## Связь с требованиями

- FR: FR-60
- US: US-22
- UC: UC-8

## Триггер

При первом открытии веб-приложения, если разрешение на push отклонено на уровне браузера.

## Структура экрана

- Модалка: иконка колокольчика, заголовок «Уведомления отключены».
- Текст: «Вы не будете получать напоминания о тренировках. Включите уведомления в настройках браузера.»
- Кнопки:
  - «Открыть настройки» (ссылка на настройки уведомлений браузера).
  - «Понятно» (закрыть).
- Показывается один раз, не блокирует функционал.

## Состояния экрана

| Состояние | Поведение |
|---|---|
| Push разрешён | Модалка не показывается |
| Push запрещён (первый раз) | Показывается модалка |
| Push запрещён (повторно) | Модалка не показывается |

## Заметки для дизайнера

- Модалка не блокирует функционал (FR-60).
- Кнопка «Открыть настройки» — обычная веб-ссылка на настройки уведомлений браузера (в разных браузерах путь может отличаться; для MVP достаточно текстовой подсказки, куда кликать).
- Показывается один раз — не надоедать пользователю.
- Информация о событиях доступна в «Мои бронирования» при следующем открытии веб-приложения (баннер для отмен скалодромом и Alert List).
