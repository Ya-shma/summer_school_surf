## 1. О продукте и цели дизайна

### 1.1. Что мы проектируем
Клиентское мобильное приложение (iOS + Android) для скалодрома «Вертикаль». Приложение даёт клиентам возможность **самостоятельно записываться на групповые тренировки**, выбирать снаряжение, отменять запись, видеть историю, оценивать инструктора и получать push-напоминания.

### 1.2. Границы скоупа
- ✅ **В скоупе:** только клиентское приложение и API для него.
- ❌ **Вне скоупа:** админка Оли, интерфейс инструктора, CRUD расписания, онлайн-оплата.

### 1.3. Цели дизайна (Design Goals)
| # | Цель | Источник |
|---|---|---|
| DG-1 | **Запись за ≤ 3 экрана** от открытия слота до подтверждения | NFR-2 |
| DG-2 | **Читаемость в зале**: крупные элементы, высокий контраст, минимум полей — руки в магнезии, шум, яркий свет | NFR-1, NFR-25 |
| DG-3 | **Нулевой порог входа**: регистрация по телефону + SMS-код, без паролей | NFR-3, FR-1 |
| DG-4 | **Предсказуемость**: клиент всегда понимает, сколько мест, есть ли прокат, почему кнопка неактивна | FR-15, FR-52, FR-55 |
| DG-5 | **Защита от ошибок**: понятные сообщения, идемпотентность, невозможность двойной брони | NFR-27, US-24, US-25 |
| DG-6 | **Доступность (a11y)**: WCAG 2.1 AA, touch-target ≥ 44pt, поддержка системного увеличения шрифта | NFR-25 |

### 1.4. Анти-цели (чего НЕ делаем)
- Онбординг с обучающими экранами — интерфейс должен быть интуитивным.
- Публичный рейтинг инструкторов — оценки приватные.
- Встроенный чат поддержки — только deep-link в Telegram.
- Онлайн-оплата — оплата офлайн, в приложении только отображение цены.
- Групповая бронь «в одной операции» — одна бронь = один участник.

## 2. Целевая аудитория и контекст использования

### 2.1. Портрет пользователя
- **Клиент скалодрома** — от 6 лет (с родителем) до взрослого.
- Пользуется приложением **с телефона** (mobile-first).
- Уровень цифровой грамотности — разный: от «только Telegram» до продвинутого.
- Мотивации: записаться на удобный слот, к «звёздному» инструктору, вместе с ребёнком; не потерять место; видеть свою статистику.

### 2.2. Контексты использования (Context of Use)
| Контекст | Условия | Следствия для дизайна |
|---|---|---|
| **В зале, перед тренировкой** | Шум, яркий свет, руки в магнезии, перчатки/скальники, спешка | Крупные тач-таргеты (≥ 44pt), высокий контраст, минимум полей, крупные CTA |
| **Дома / в транспорте** | Спокойная обстановка, одна рука свободна | Можно листать ленту, фильтровать, читать карточки |
| **Сразу после push-напоминания** | Клик по пушу → открытие конкретного экрана | Глубокие ссылки (deep-link) должны вести точно в карточку / бронь |
| **Нестабильная сеть в ангаре** | Плохой Wi-Fi / мобильный интернет | Offline-режим для чтения, понятные Retry-сообщения, идемпотентность |

## 3. Дизайн-принципы

1. **Mobile-first и «заловый» UI** — всё, что можно, делаем крупным и контрастным. Предположим, что у пользователя одна рука в магнезии.
2. **Одно действие — одна CTA** — на экране всегда одна главная кнопка.
3. **Состояние видно сразу** — свободные места, прокат, доступ к верёвкам, блокировка — всё отображается в карточке/ленте, без лишних кликов.
4. **Сервер — источник истины** — клиент никогда не «угадывает» доступность, а получает её из API (включая `can_cancel`, `rope_access`, `is_blocked`).
5. **Не наказываем за незнание** — если кнопка неактивна, рядом всегда есть пояснение, почему.
6. **Минимум модальных окон** — только там, где есть юридическое значение (согласие на ПДн, согласие с ТБ).

## 4. Визуальный язык и брендинг

> Бренд скалодрома не задан. Ниже — рекомендации, которые дизайнер может адаптировать.

### 4.1. Настроение
- **Спортивное, утилитарное, дружелюбное.** Без «премиальности» и без «детскости».
- Ощущение «свой клуб»: тёплое, но функциональное.

### 4.2. Цветовая палитра (рекомендация)
| Роль | Цвет | Назначение |
|---|---|---|
| Primary | Глубокий терракотовый / охра (`#C65D3A`) | CTA, акценты, активные состояния |
| Background | Тёплый светло-серый (`#F6F3EE`) | Фон экранов |
| Surface | Белый (`#FFFFFF`) | Карточки, модалки |
| Text / Primary | Почти чёрный (`#1A1A1A`) | Основной текст |
| Text / Secondary | Тёплый серый (`#6B655C`) | Подписи, второстепенный текст |
| Success | Приглушённый зелёный (`#3E8A5C`) | Подтверждения, «свободно» |
| Warning | Приглушённый жёлтый (`#D9A441`) | Предупреждения, Alert List |
| Error | Приглушённый красный (`#B5413A`) | Ошибки, «мест нет», блокировки |
| Disabled | Серый (`#BDBDBD`) | Неактивные кнопки |

> Все пары должны проходить WCAG AA (контраст ≥ 4.5:1 для обычного текста, ≥ 3:1 для крупного).

### 4.3. Типографика
- **Шрифт:** системный (San Francisco / Roboto) — быстро, нативно, поддерживает Dynamic Type.
- **Заголовки:** SemiBold, 20–24pt.
- **Основной текст:** Regular, 16–17pt (не меньше — читаемость в зале).
- **Подписи:** Regular, 13–14pt.
- **Минимальный размер:** 13pt (даже для подписей).

### 4.4. Иконки
- Линейные, 24×24, толщина 1.5–2px.
- Семантика должна дублироваться текстом там, где она неочевидна (не полагаемся только на иконку — NFR-25).

### 4.5. Скругления и тени
- Карточки: 12–16px.
- Кнопки: 12px.
- Тени — мягкие, едва заметные (elevation 1–2).

## 6. Ключевые пользовательские потоки (User Flows)

### 6.1. Первый запуск → первая запись
Открытие приложения
→ Экран входа
→ Ввод телефона
→ Галочка «Согласен на обработку ПДн» (FR-50)
→ «Получить код»
→ SMS OTP → ввод кода
→ Запрос имени (обязательно) + возраст/ДР (опционально)
→ Главный экран «Расписание»
→ Тап по слоту → карточка
→ «Записаться» → модалка согласия с ТБ (FR-51)
→ Выбор «свои скальники / прокат»
→ Подтверждение → успех → «Мои бронирования»

### 6.2. Постоянный клиент → приоритетная запись
Расписание
→ Слот, который откроется для обычных через N дней
→ Постоянный клиент видит CTA «Записаться» активной
→ Обычный клиент видит пояснение «Откроется через N дней» + неактивная CTA (FR-52)

### 6.3. Заполненный слот → Alert List
Карточка заполненного слота
→ CTA «Уведомить, если появится место» (FR-53)
→ Нажатие → состояние «Вы в списке ожидания»
→ Освобождение места → push
→ Клик по push → карточка слота → запись

### 6.4. Отмена записи
«Мои бронирования» → детали брони
→ Если can_cancel = true → CTA «Отменить» активна
→ Если can_cancel = false → пояснение политики 6ч + CTA «Написать Оле»

### 6.5. Оценка инструктора
Push «Оцените инструктора» (через 1ч после тренировки)
→ Deep-link в экран оценки
→ 1–5 звёзд → «Отправить»
→ Успех → возврат в «Мои бронирования»

## 7. Детальное описание экранов

### 7.1. Экран входа / регистрации
**Цель:** максимально быстро ввести клиента в сессию.  
**Связь:** FR-1, FR-2, FR-43, FR-50, NFR-3, NFR-19, US-1, UC-1.

**Структура:**
1. Логотип «Вертикаль» + заголовок «Войти по телефону».
2. Поле ввода телефона с маской `+7 (___) ___-__-__`.
3. Чекбокс «Согласен на обработку персональных данных» + ссылка на политику.
4. CTA «Получить код» (активна только при валидном телефоне + галочке).
5. Мелкий текст: «Войдёте по SMS-коду. Это бесплатно для вас».

**Состояния:**
- **Пустое:** поле не заполнено → CTA disabled.
- **Отправка кода:** loading-индикатор на кнопке.
- **Ошибка отправки SMS:** тост «Не удалось отправить код, попробуйте позже» (E1 UC-1).
- **Rate-limit:** сообщение «Слишком много попыток, попробуйте через N минут» (E3 UC-1).

**Экран ввода OTP:**
1. Заголовок «Код отправлен на +7 *** *** ** 67».
2. 4–6 полей для кода (авто-фокус, авто-чтение из SMS на Android).
3. Таймер «Отправить повторно через 00:30» → становится кнопкой.
4. Счётчик неверных попыток: после 3 — экран блокировки на 15 минут с таймером (NFR-19).

**Экран заполнения профиля (только для новых):**
1. Имя (обязательно).
2. Возраст (опционально, числовое поле).
3. День рождения (опционально, date-picker).
4. CTA «Продолжить».

---

### 7.2. Главный экран — «Расписание»
**Цель:** показать слоты на 7 дней и дать быстро найти нужный.  
**Связь:** FR-9, FR-38, FR-9a, FR-52, FR-53, US-2, US-3, US-4, US-5, US-16, UC-2.

**Структура:**
1. **Шапка:**
   - Заголовок «Расписание».
   - Иконка фильтра (справа) → открывает шторку фильтров.
   - Активные фильтры — чипы под шапкой (с крестиком для сброса).
2. **Горизонтальный скролл дат** (7 дней): «Сегодня / Завтра / Ср / Чт / Пт / Сб / Вс» + кнопка «Ещё» (кастомный диапазон).
3. **Лента слотов** — вертикальный список, сгруппированный по дням.
4. **Empty state:** «Пока нет доступных тренировок» + кнопка «Сбросить фильтры».

**Карточка слота в ленте (компактная):**
- Время старта (крупно, 24pt, слева).
- Формат: чип «Новички» / «Опытные» (цветовой код).
- Инструктор: имя мелким текстом.
- Справа: `X / N мест` (например, `5 / 8`) + цена `1500 ₽`.
- Индикаторы:
  - 🔴 `Мест нет` (если `free_seats = 0`).
  - ⚠️ `Отменена скалодромом` (если `cancelled_by_gym`).
  - 🔒 `Откроется через N дней` (если слот ещё не открыт для категории клиента, FR-52).

**Состояния карточки:**
| Состояние | CTA | Поведение |
|---|---|---|
| Есть места, слот открыт | «Записаться» (активна) | → карточка слота |
| Мест нет | «Уведомить, если появится место» | → Alert List (FR-53) |
| Отменена скалодромом | нет CTA, плашка с причиной | → детали с причиной |
| Слот ещё не открыт | неактивна + пояснение «Откроется через N дней» | тап не ведёт в запись |
| Клиент заблокирован | неактивна + «Вы заблокированы до N» | → детали с датой |

**Offline-состояние (NFR-24):**
- Лента из кэша + баннер сверху «Данные могли устареть · Обновить».
- CTA записи недоступны.

### 7.3. Шторка фильтров
**Цель:** быстро отфильтровать слоты.  
**Связь:** FR-38, US-3, UC-2.

**Структура:**
1. **Период:** сегмент-контрол «Сегодня / Эта неделя / Выходные / Кастомный» + date-picker для кастомного.
2. **Формат:** чипы «Новички» / «Опытные» (мультивыбор, OR внутри группы).
3. **Инструктор:** список чипов с именами (скрывается, если справочник пуст — A4 UC-2).
4. **Опция:** «Только свободные» (toggle).
5. Кнопки «Сбросить» и «Применить».

**Семантика:** внутри группы — OR, между группами — AND (FR-38).

---

### 7.4. Карточка слота (детальная)
**Цель:** дать всю информацию для принятия решения и записаться.  
**Связь:** FR-9a, FR-10, FR-11, FR-13, FR-14, FR-15, FR-51, FR-54, FR-55, US-4, US-6, US-7, US-8, US-9, US-10, UC-2, UC-3.

**Структура:**
1. **Заголовок:** дата, время, формат (бейдж).
2. **Инструктор:** имя + фото (если есть).
3. **Статус мест:** прогресс-бар `5 / 8 мест` + `12 / 20 комплектов проката свободно`.
4. **Цена:** `1500 ₽` (фиксированно) + стоимость проката (если выбран).
5. **Адрес:** текстом (без карты — по ответам Оли).
6. **Чек-лист «Что взять»:** короткий текст.
7. **Ссылка «Правила и техника безопасности»** → внешний URL.
8. **CTA-зона (внизу, зафиксирована):**
   - Активная: «Записаться · 1500 ₽».
   - Неактивная с пояснением (гейткинг / блокировка / мест нет / отменён).

**Варианты CTA-состояний:**
| Состояние | Текст CTA | Пояснение |
|---|---|---|
| Можно записаться | «Записаться · 1500 ₽» | — |
| Нужен прокат, но фонда нет | «Проката нет» (disabled) | «Выберите свои скальники» |
| Нет флага rope_access на опытном | «Недоступно» (disabled) | «Сначала пройдите вводную тренировку для новичков» (FR-55) |
| Клиент заблокирован | «Запись недоступна» (disabled) | «Вы заблокированы до 12.07. Обратитесь к владельцу» (FR-56) |
| Мест нет | «Уведомить, если появится место» | → Alert List |
| Слот отменён скалодромом | нет CTA | причина отмены сверху |
| Слот уже стартовал | нет CTA | «Тренировка началась» |
| Слот ещё не открыт | «Откроется через N дней» (disabled) | для непостоянных (FR-52) |

### 7.5. Экран записи (модалка подтверждения)
**Цель:** подтвердить бронь за ≤ 3 шага от карточки слота.  
**Связь:** FR-10, FR-11, FR-51, FR-54, FR-45, US-6, US-7, US-8, US-9, UC-3.

**Сценарий (≤ 3 экрана):**
1. **Экран 1 — параметры брони:**
   - Слот (время, формат, инструктор) — read-only.
   - **Сегмент «Снаряжение»**: «Свои скальники» / «Нужно прокатное снаряжение» (FR-11).
   - **Опция «Записать ребёнка»** (toggle, FR-54) → при включении появляется поле «Возраст ребёнка» (число, ≥ 6).
   - Итоговая цена: `1500 ₽` + `X ₽ прокат` (если выбран).
   - CTA «Подтвердить запись».
2. **Экран 2 (только при первой записи в жизни) — согласие с ТБ (FR-51):**
   - Модалка с текстом правил.
   - Чекбокс «Ознакомлен с правилами и техникой безопасности».
   - CTA «Продолжить» (disabled без галочки).
   - Флаг сохраняется в профиле навсегда.
3. **Экран 3 — успех:**
   - Галочка, «Вы записаны!».
   - Краткая сводка: дата, время, формат, снаряжение, цена.
   - Кнопки: «В мои бронирования» / «Закрыть».

**Защита от двойной брони:**
- При каждом подтверждении клиент передаёт `Idempotency-Key` (UUID v4). Повтор при таймауте — с тем же ключом (NFR-27, US-24).

**Ошибки и UX-реакции (матрица из UC-3):**
| Код | HTTP | UX |
|---|---|---|
| `slot_full` | 409 | «Места закончились» + обновить карточку |
| `double_booking` | 409 | «Вы уже записаны» → переход в «Мои бронирования» |
| `rope_access_required` | 403 | «Сначала пройдите вводную тренировку» |
| `client_blocked` | 403 | «Вы заблокированы до N» |
| `rental_unavailable` | 409 | «Проката нет, выберите свои скальники» |
| `slot_cancelled` | 410 | «Тренировка отменена» |
| `slot_started` | 422 | «Тренировка уже началась» |
| `child_age_invalid` | 400 | «Возраст ребёнка должен быть ≥ 6 лет» |

### 7.6. Экран «Мои бронирования»
**Цель:** видеть активные и прошедшие записи, управлять ими.  
**Связь:** FR-35a, FR-16, FR-17, FR-18, FR-46, US-12, US-13, US-14, US-15, UC-4.

**Структура:**
1. **Сегмент-контрол:** «Предстоящие» / «Прошедшие».
2. **Список карточек броней**, отсортированный по `slot.start_at`.
3. **Empty state:** «У вас пока нет записей» + CTA «Посмотреть расписание».

**Карточка брони:**
- Дата/время, формат, инструктор.
- Статус-бейдж:
  - `booked` — синий/primary
  - `attended` — зелёный, «Посещено»
  - `cancelled_by_client` — серый, «Отменено вами»
  - `cancelled_late` — жёлтый, «Поздняя отмена»
  - `no_show` — жёлтый, «Неявка»
  - `cancelled_by_gym` — красный, «Отменено скалодромом» + причина
- Снаряжение: иконка + текст.
- Возраст (если детская бронь): «Ребёнок, 8 лет».
- CTA:
  - `booked` + `can_cancel=true` → «Отменить».
  - `booked` + `can_cancel=false` → «Написать Оле» (deep-link в Telegram).
  - `cancelled_by_gym` → «Выбрать другое время» (открывает расписание с фильтром по формату, FR-46).
  - `attended` + нет оценки + прошло ≥ 1ч → «Оценить инструктора» (FR-57).

**Баннер для отмен скалодромом (если push не дошёл):**
- Вверху списка: «N тренировок отменено скалодромом» → раскрытие со списком.

---

### 7.7. Экран отмены брони
**Цель:** безопасно отменить запись с пониманием последствий.  
**Связь:** FR-16, FR-17, FR-18, US-13, US-14, UC-4.

**Сценарий:**
1. Тап «Отменить» в карточке брони → модалка подтверждения.
2. Текст: «Отменить запись на [дата, время]? Место и прокат освободятся.»
3. Если до старта < 6ч → модалка не показывается, кнопка сразу неактивна, под ней пояснение: «Отмена возможна не позднее чем за 6 часов до начала. Если нужно отменить позже — напишите Оле».
4. CTA «Отменить запись» (деструктивная, красная).
5. Успех → тост «Запись отменена» + обновление списка.

---

### 7.8. Экран оценки инструктора
**Цель:** быстро оценить после тренировки.  
**Связь:** FR-57, FR-58, US-17, UC-6.

**Структура:**
1. Заголовок: «Как прошла тренировка?»
2. Инструктор: имя, дата тренировки.
3. 5 звёзд (тап по звезде — выбор).
4. CTA «Отправить оценку» (активна после выбора).
5. Успех → тост «Спасибо за оценку!» → возврат в «Мои бронирования».

**Ограничения:**
- Нет текстовых комментариев (FR-58).
- Оценка видна только владельцу (не показываем клиенту «средний рейтинг»).
- Доступно только для статуса `attended` (FR-57).

---

### 7.9. Экран «Профиль»
**Цель:** данные клиента, бейджи, быстрые действия.  
**Связь:** FR-47, FR-49, FR-59, FR-16a, US-18, US-19, US-20, UC-7.

**Структура:**
1. **Шапка:**
   - Имя, телефон (маскирован: `+7 *** *** ** 67`, NFR-20).
   - Бейдж «Постоянный клиент» (если `is_permanent=true`, FR-59).
   - Возраст, день рождения (если указаны).
2. **Блок «Статистика»:**
   - «Пройдено тренировок: N» (счётчик `attended`).
   - Прогресс до постоянного клиента (если ещё не): «До постоянного клиента: 10 − N».
3. **Блок «Документы»:**
   - «Правила скалодрома и ТБ» → внешний URL.
   - «Политика конфиденциальности» → внешний URL.
4. **Блок «Связь»:**
   - «Написать Оле» → deep-link `tg://` в Telegram (FR-16a).
5. **Блок «Аккаунт»:**
   - «Редактировать данные» → экран редактирования (имя, телефон, возраст, ДР).
   - «Выйти» → модалка подтверждения → очистка токенов из Keychain/Keystore (FR-49, NFR-18).

---

### 7.10. Экран редактирования профиля
**Цель:** обновить данные.  
**Связь:** FR-47, US-19.

**Поля:**
- Имя (обязательно).
- Телефон (только просмотр, смена — через Telegram + ручная привязка владельцем).
- Возраст (опционально).
- День рождения (опционально).

**CTA:** «Сохранить» → при успехе тост «Данные обновлены».

---

### 7.11. Предупреждение о push (FR-60, US-22)
**Цель:** один раз объяснить клиенту, что он не будет получать напоминания.  
**Триггер:** при первом открытии приложения, если разрешение на push отклонено на уровне ОС.

**Структура:**
- Модалка: иконка колокольчика, заголовок «Уведомления отключены».
- Текст: «Вы не будете получать напоминания о тренировках. Включите уведомления в настройках телефона.»
- Кнопки: «Открыть настройки» (системный deep-link) / «Понятно» (закрыть).
- **Показывается один раз**, не блокирует функционал.

## 9. Микровзаимодействия и состояния

### 9.1. Загрузка
- **Лента слотов:** skeleton-карточки.
- **Карточка слота:** skeleton + актуальный запрос при каждом открытии (NFR-31).
- **Кнопка:** loading-индикатор внутри, блокировка повторных нажатий.

### 9.2. Успех
- Короткий тост (2 сек) или экран успеха (для записи).
- Haptic feedback (light) на подтверждение.

### 9.3. Ошибка
- **Сетевая:** баннер «Нет соединения» + кнопка «Повторить».
- **Бизнес-ошибка (4xx):** понятный текст + действие (обновить / изменить выбор).
- **Серверная (5xx):** «Что-то пошло на стороне сервера. Повторите через минуту» + Retry.
- **Неопределённый результат** (таймаут при мутации): автоматический retry с тем же `Idempotency-Key` (NFR-27).

### 9.4. Offline
- Чтение из кэша с плашкой «Данные могли устареть».
- Мутации (запись, отмена, оценка, Alert List) заблокированы.

## 10. Доступность (a11y)

**Обязательные требования (NFR-25):**
- Контраст текста ≥ 4.5:1 (WCAG AA).
- Touch-target ≥ 44×44pt для всех интерактивных элементов.
- Поддержка системного увеличения шрифта (Dynamic Type) без потери функциональности.
- Все интерактивные элементы имеют `accessibilityLabel` для VoiceOver / TalkBack.
- Смысл не передаётся только цветом (дублируем иконкой / текстом).
- Фокус-стейт для клавиатурной навигации (iPad / Android с клавиатурой).

**Проверки перед релизом:**
- Пройти скринридером все экраны.
- Проверить на шрифте +3 размера.
- Проверить контраст всех пар цветов.

## 12. Что НЕ делаем в V1 (явно)

- ❌ Онбординг с обучающими экранами.
- ❌ Публичный рейтинг инструкторов.
- ❌ Текстовые комментарии к оценкам.
- ❌ Онлайн-оплата.
- ❌ SMS/WhatsApp-каналы уведомлений.
- ❌ Встроенный чат поддержки.
- ❌ Групповые брони «в одной операции».
- ❌ Перенос записи как единая операция (есть отмена + новая запись).
- ❌ Расширенные механики лояльности (скидки, бесплатные тренировки).
- ❌ Карта проезда (только текстовый адрес).
- ❌ Список участников слота (приватность).

## 13. Критерии приёмки дизайна

Дизайн считается готовым к прототипированию, если:
1. **Нарисованы все экраны из §7** в трёх состояниях: пустое / загрузка / данные.
2. **Проработаны все состояния CTA** из §7.4 (таблица вариантов карточки слота).
3. **Соблюдены a11y-требования** (§10): контраст, touch-target, Dynamic Type.
4. **Соблюдён лимит ≤ 3 экранов** до подтверждения записи (DG-1).
5. **Все экраны трассируются** к FR/US/UC (§11).
6. **Собраны компонентные состояния** (§8) и микровзаимодействия (§9).
7. **Спроектированы deep-link'и** из push-уведомлений (UC-8).
