Дизайн-бриф: клиентское веб-приложение скалодрома «Вертикаль» (V1 / MVP)
Статус: черновик для передачи в дизайн / прототипирование
Связанные артефакты: `brief-climbing.md`, `domain-description.md`, `customer-questions.md`, `answers.md`, `business-requirements.md`, `functional-requirements.md`, `non-functional-requirements.md`, `user-stories.md`, `use-cases.md`

1. Контекст и цели
1.1. Продукт и платформа
Клиентское веб-приложение (responsive, mobile-first), доступное через браузер на телефоне и десктопе. Не требует установки из сторов, работает по URL.

2. Аудитория и контекст использования
2.1. Целевая аудитория
Клиенты скалодрома, 6+ лет, разный уровень цифровой грамотности.
2.2. Контекст
Пользуется через браузер (с телефона или десктопа): в зале (шум, яркий свет, возможны следы магнезии на руках) и дома/в транспорте.

4. Визуальный язык
4.1. Цветовая палитра
| Роль | Цвет | Назначение |
|---|---|---|
| Primary | `#C65D3A` (терракотовый/охра) | CTA, акценты |
| Background | `#F6F3EE` (тёплый светло-серый) | Фон экранов |
| Surface | `#FFFFFF` | Карточки, модалки |
| Text/Primary | `#1A1A1A` | Основной текст |
| Text/Secondary | `#6B655C` | Подписи |
| Success | `#3E8A5C` | Подтверждения |
| Warning | `#D9A441` | Предупреждения |
| Error | `#B5413A` | Ошибки |
| Disabled | `#BDBDBD` | Неактивные кнопки |
Все пары должны проходить WCAG AA (контраст ≥ 4.5:1 для обычного текста, ≥ 3:1 для крупного).

4.3. Типографика
Шрифт: системный (San Francisco / Roboto / системный шрифт браузера) — быстро, нативно, поддерживает системное увеличение шрифта в браузере.
Заголовки: SemiBold, 20–24pt.
Основной текст: Regular, 16–17pt (не меньше — читаемость в зале).
Подписи: Regular, 13–14pt. Минимальный размер: 13pt (даже для подписей).
Отступы: 4px grid. Карточки: 12–16px. Кнопки: 12px. Тени — мягкие, едва заметные (elevation 1–2).

Адаптивная навигация: Bottom Tab Bar на мобильных / Top navigation на десктопе — 3 раздела: «Расписание», «Мои бронирования», «Профиль».
Вверх по стеку — кнопка back браузера / back-gesture на мобильных.
Модалки — только для согласий (ПДн, ТБ) и фильтров.
Deep-link из push — открывает нужный экран внутри стека (карточка слота / детали брони / экран оценки).

8. Микровзаимодействия и состояния
8.3. Ошибки
Сетевая: баннер «Нет соединения» + кнопка «Повторить».
Бизнес-ошибка (4xx): понятный текст + действие (обновить / изменить выбор). Тексты и действия для бизнес-ошибок см. в матрице ошибок UC-3 (раздел «Исключения / ошибки»).
Серверная (5xx): «Что-то пошло на стороне сервера. Повторите через минуту» + Retry.
Неопределённый результат (таймаут при мутации): автоматический retry с тем же `Idempotency-Key` (NFR-27).
8.4. Offline
Чтение из кэша с плашкой «Данные могли устареть».
Мутации (запись, отмена, оценка, Alert List) заблокированы.

9. Доступность (a11y)
Контраст ≥ WCAG AA (4.5:1 текст, 3:1 крупный).
touch/click-target ≥ 44px.
Поддержка системного увеличения шрифта в браузере без потери верстки.
Скринридеры: семантическая разметка, label для всех интерактивных элементов, поддержка NVDA, VoiceOver, TalkBack.
Смысл не передаётся только цветом.

10. Трассировка
Каждый экран и компонент обязан трассироваться к FR/US/UC из аналитики. См. `user-stories.md`, `use-cases.md`, `functional-requirements.md`.

11. Готовность к прототипированию
Дизайн считается готовым к прототипированию, если:
- Нарисованы все экраны из §5.3 в трёх состояниях: пустое / загрузка / данные.
- Проработаны все состояния CTA из `screen-04-slot-detail.md` (таблица вариантов карточки слота).
- Соблюдены a11y-требования (§9): контраст, touch/click-target ≥ 44px, поддержка системного увеличения шрифта.
- Соблюдён лимит ≤ 3 экранов до подтверждения записи (DG-1).
- Все экраны трассируются к FR/US/UC (§10).
- Собраны компонентные состояния (§7) и микровзаимодействия (§8).
- Спроектированы deep-link'и из push-уведомлений (UC-8).
