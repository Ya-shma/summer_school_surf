**О нумерации требований.** ID требований (FR-NN) сохраняются стабильными для трассировки. В нумерации есть пропуски — это следствие осознанного сужения скоупа (часть ранее заведённых требований вынесена в развитие/Phase 2 или относится к существующей инфраструктуре). Нумерация намеренно не нормализуется, чтобы не ломать связи в таблице-источнике и смежных документах. Новые требования продолжают сквозную схему FR-NN без переиспользования занятых номеров.

## Границы скоупа (клиентское приложение + API)

Скоуп — клиентское мобильное приложение и API для него. Функции инструктора, владельца и администратора обеспечиваются существующей инфраструктурой и здесь не описываются.

**Следствия для скоупа:**

- Слоты (тренировки) приходят из существующего бэкенда через API. Экранов создания и редактирования расписания в клиентском приложении нет.
- Форматы и инструкторы доступны клиенту как справочные (read-only) данные из API — клиент видит их в карточке слота, но не управляет ими.
- Флаг «допущен к верёвкам», статус «постоянный клиент», счётчик нарушений, блокировки клиентов — ведутся в существующей инфраструктуре; приложение лишь потребляет их через API.

**Граница со серверной интеграцией.** Детали серверной реализации брони (транзакции, SLA, маппинг полей) — вне скоупа клиентского приложения. Существующий бэкенд выступает black-box источником истины; атомарность операций и гарантия «0 двойных броней» обеспечиваются на его стороне. Граница между приложением и бэкендом закрывается контрактом API; клиент полагается на коды ответов и значения полей из ответа (включая `max_seats`, `free_rental_boards`, `price_total`, `rope_access`, `is_blocked`, `is_permanent`).

**Данные существующей инфраструктуры (R-015).** Проект учебный/тестовый, легаси-данных нет. Каноническая схема — контракт API (01-analysis/api/): бэкенд по условию предоставляет все поля контракта (`rental_price`, `address`, `free_rental_boards`, `rope_access`, `is_permanent` и пр.). Требований к миграции, backfill и дозаполнению исторических слотов не формулируем — это вне скоупа клиентского приложения.

## Вне скоупа

Функции инструктора, владельца/администратора, расписание-CRUD, онлайн-оплата, автоматический учёт погоды, расширенные механики лояльности (скидки, бесплатные тренировки), публичный рейтинг инструкторов, текстовые комментарии к оценкам, SMS/WhatsApp-каналы, встроенный чат поддержки, онбординг с обучающими экранами, групповые брони (запись нескольких человек в одной операции) — обеспечиваются существующей инфраструктурой или относятся к развитию (V2+). В клиентском приложении и его API не реализуются.
