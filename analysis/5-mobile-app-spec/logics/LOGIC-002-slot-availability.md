LOGIC-002 — Расчёт доступности слота
Цель
Определить, может ли клиент записаться на слот.

Связь с требованиями
FR: FR-15, FR-52, FR-55, FR-56
US: US-5, US-10, US-11
UC: UC-2, UC-3

Входные данные
Данные слота из API: `free_seats`, `free_rental_boards`, `opens_at`, `cancelled_by_gym`, `start_at`.
Данные профиля клиента: `rope_access`, `is_blocked`, `blocked_until`, `is_permanent`.

Выходные данные
Состояние CTA (активна / неактивна).
Текст пояснения.
Доступные действия.

Алгоритм
Все флаги (`can_book`, `rope_access_required`, `is_blocked`, `opens_at`) приходят с сервера — веб-приложение не вычисляет их само.
Веб-приложение только отображает значения из ответа API.
Приоритетная запись постоянных (FR-52): слоты открываются за 2 дня для постоянных, за 1 день для остальных.

Состояния CTA
| Состояние | Текст CTA | Пояснение |
|---|---|---|
| Можно записаться | «Записаться · 1500 ₽» | — |
| Нужен прокат, но фонда нет | «Проката нет» (disabled) | «Выберите свои скальники» |
| Нет флага rope_access | «Недоступно» (disabled) | «Сначала пройдите вводную тренировку для новичков» |
| Клиент заблокирован | «Запись недоступна» (disabled) | «Вы заблокированы до N» |
| Мест нет | «Уведомить, если появится место» | → Alert List |
| Слот отменён | нет CTA | причина отмены сверху |
| Слот стартовал | нет CTA | «Тренировка началась» |
| Слот ещё не открыт | «Откроется через N дней» (disabled) | для непостоянных |
