LOGIC-004 — Правило 6 часов отмены
Цель
Определить, можно ли отменить запись.

Связь с требованиями
FR: FR-16, FR-17, FR-18
US: US-13, US-14
UC: UC-4

Входные данные
`can_cancel`, `cancel_deadline` из ответа API.

Выходные данные
Состояние CTA «Отменить» (активна / неактивна).
Текст пояснения.

Алгоритм
Сервер — источник истины по времени и статусу отмены (FR-17).
Граница «ранней» отмены — ровно за 6 часов до старта включительно.
Веб-приложение только отображает значения из ответа API.

Состояния CTA
| Состояние | Поведение |
|---|---|
| can_cancel = true | Кнопка «Отменить» активна → BS-003 |
| can_cancel = false | Кнопка «Отменить» неактивна + пояснение «Отмена возможна не позднее чем за 6 часов» + CTA «Написать Оле» (FR-16a) |

Применяется в
SCR-006 (Мои бронирования).
SCR-007 (Детали брони).
BS-003 (Подтверждение отмены).
