## 3. Навигация

- **На мобильном:** Bottom Tab Bar (АЗ): 3 вкладки — «Расписание», «Мои бронирования», «Профиль».
- **На десктопе:** Top navigation bar или sidebar с теми же пунктами.

**Ключевые переходы:**
- SCR-001 → SCR-002 (успех входа) → SCR-003 (тап/клик по слоту) → SCR-004 (записаться) → BS-002 (первая запись, согласие с ТБ) → SCR-005 (успех) → SCR-006 (в «Мои бронирования»).
- SCR-006 → SCR-007 (детали брони) → BS-003 (подтверждение отмены) → SCR-006 (обновление списка).
- SCR-003 (заполненный слот) → Alert List (LOGIC-009) → push → SCR-003 (повторное открытие).
- Deep-link из браузерного push → SCR-003 / SCR-007 / SCR-008 в зависимости от типа уведомления.

**Навигация вверх:** системный back браузера / back-gesture на мобильных устройствах.

## 4. Соглашения

**Платформа.** Клиентское веб-приложение (responsive, mobile-first), доступное через браузер на телефоне и десктопе (NFR-4). Целевые браузеры: Chrome, Safari, Firefox, Edge (последние 2 версии); мобильные: iOS Safari 15+, Android Chrome (современные версии). Опора на системные возможности браузера: браузерный push (Web Push API, LOGIC-006, FR-33) и защищённое хранилище (HttpOnly cookies / secure localStorage, LOGIC-001, NFR-18).

**API.** Все запросы REST. Сводная таблица эндпоинтов — `api-sequence-diagrams.md` §10. Все мутирующие эндпоинты требуют заголовок `Idempotency-Key` (NFR-27, LOGIC-008).

**Числа не хардкодятся.** Потолки мест по формату (8/16), прокатный фонд (20), цена тренировки (1500 ₽), тариф проката, дата окончания блокировки, дата открытия слота для категории клиента — всё приходит из API.

**Сервер — источник истины.** Флаги `can_cancel`, `rope_access`, `is_blocked`, `is_permanent`, `opens_at`, `free_seats`, `free_rental_boards`, `price_total` приходят из API и отображаются как есть (NFR-31, LOGIC-002, LOGIC-003, LOGIC-004).

**Границы скоупа (R-004).** Существующий бэкенд — black-box источник истины. Транзакционность, атомарность, гарантия «0 двойных броней» — на стороне бэкенда.

**Время (NFR-29).** API принимает/возвращает время в UTC (ISO 8601). Веб-приложение отображает в локальной часовой зоне (через `Intl.DateTimeFormat` / системную таймзону браузера).

**Offline (NFR-24).** Чтение — из кэша (через Service Worker / Cache API) с пометкой «данные могли устареть». Мутации офлайн запрещены.

**Приватность (NFR-20).** В UI и логах — только маскированный телефон (`+7 *** *** ** 67`). OTP-коды и токены не логируются.

**A11y (NFR-25).** WCAG 2.1 AA: контраст ≥ 4.5:1, touch/click-target ≥ 44×44px, поддержка системного увеличения шрифта в браузере, `aria-label` / `accessibilityLabel` для скринридеров (NVDA, VoiceOver, TalkBack), смысл не передаётся только цветом.

**Типы компонентов:**
- Экран (SCR-NNN) — полноэкранная навигационная единица, входит в стек.
- Bottom Sheet (BS-NNN) — модальная шторка снизу (на мобильном) / центральный dialog (на десктопе).
- Modal Dialog — центральный диалог.

## 6. Что НЕ входит в ТЗ (явно)

Следующее находится вне скоупа клиентского веб-приложения и не описывается в ТЗ:

❌ Админка Оли, интерфейс инструктора, CRUD расписания.
❌ Онлайн-оплата (оплата офлайн, 1500 ₽ фиксированно).
❌ SMS/WhatsApp-каналы уведомлений (только браузерный push через Web Push API).
❌ Встроенный чат поддержки (только веб-ссылка `https://t.me/...` в Telegram, FR-16a).
❌ Публичный рейтинг инструкторов (оценки приватные, FR-58).
❌ Текстовые комментарии к оценкам (только звёзды 1–5).
❌ Групповые брони «в одной операции» (одна бронь = один участник).
❌ Перенос записи как единая операция (реализуется как отмена + новая запись).
❌ Онбординг с обучающими экранами.
❌ Расширенные механики лояльности (скидки, бесплатные тренировки) — V2.
❌ Карта проезда (только текстовый адрес).
❌ Список участников слота (приватность).

Конец основного файла ТЗ. Детальные ТЗ экранов — в файлах `SCR-NNN-*.md` и `BS-NNN-*.md`; переиспользуемых логик — в файлах `LOGIC-NNN-*.md`.
