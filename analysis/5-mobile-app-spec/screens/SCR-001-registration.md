SCR-001 — Регистрация / Вход

Цель
Максимально быстро ввести клиента в сессию.

Связь с требованиями
FR: FR-1, FR-2, FR-43, FR-50
NFR: NFR-3 (лёгкая регистрация), NFR-18 (сессия JWT), NFR-19 (антифрод OTP)
US: US-1
UC: UC-1

Применяемые логики
LOGIC-001 (OTP-авторизация)
LOGIC-007 (Паттерн состояний экрана)
LOGIC-010 (Матрица ошибок)

Структура
Шаг 1. Ввод телефона
Логотип «Вертикаль» + заголовок «Войти по телефону».
Поле ввода телефона с маской `+7 (___) ___-__-__`.
Чекбокс «Согласен на обработку персональных данных» + ссылка на политику (FR-50).
CTA «Получить код» (активна при валидном телефоне + галочке).
Мелкий текст: «Войдёте по SMS-коду. Это бесплатно для вас».
Ссылка на политику конфиденциальности — кликабельный текст.

Шаг 2. Ввод OTP
Заголовок «Код отправлен на +7 *** *** ** 67».
4–6 полей для кода (авто-фокус, WebOTP API для авто-чтения из SMS — ограниченная поддержка в браузерах).
Таймер «Отправить повторно через 00:30» → становится кнопкой.
Счётчик неверных попыток: после 3 — экран блокировки на 15 минут (NFR-19).

Шаг 3. Заполнение профиля (только для новых)
Имя (обязательно).
Возраст (опционально, числовое поле).
День рождения (опционально, date-picker).
CTA «Продолжить».

Состояния экрана
Шаг 1. Ввод телефона
| Состояние | Поведение |
|---|---|
| Пустое | CTA disabled |
| Отправка кода | Loading-индикатор на CTA |
| Ошибка отправки SMS (E1 UC-1) | Тост «Не удалось отправить код, попробуйте позже» |
| Rate-limit (E3 UC-1) | «Слишком много попыток, попробуйте через N минут» |
| Нет сети (E4 UC-1) | Баннер «Нет соединения» + Retry |

Шаг 2. Ввод OTP
| Состояние | Поведение |
|---|---|
| Неверный код (E2 UC-1) | «Неверный код, попробуйте ещё раз» |
| 3 неверных попытки | Экран блокировки на 15 минут с таймером |
| Успех (существующий клиент) | → SCR-002 |
| Успех (новый клиент) | → Шаг 3 |

Навигация
Входящая: запуск веб-приложения без активной сессии.
Исходящая: SCR-002 (успех входа).
