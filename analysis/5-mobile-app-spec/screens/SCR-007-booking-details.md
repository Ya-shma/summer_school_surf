# SCR-007 — Детали брони

## Цель
Безопасно отменить запись с пониманием последствий.

## Связь с требованиями
- **FR:** FR-16, FR-17, FR-18, FR-16a
- **US:** US-13, US-14
- **UC:** UC-4

## Применяемые логики
- LOGIC-004 (Правило 6 часов отмены)
- LOGIC-008 (Идемпотентность мутаций)
- LOGIC-010 (Матрица ошибок)

---

## Сценарий
1. Отображение деталей брони (дата, время, формат, инструктор, снаряжение, цена).
2. CTA «Отменить» — состояние зависит от `can_cancel` (LOGIC-004).
3. При тапе «Отменить» → BS-003 (подтверждение).
4. Успех → тост «Запись отменена» + возврат в SCR-006.

## Состояния CTA (LOGIC-004)
| Состояние | Поведение |
|---|---|
| Ранняя отмена (`can_cancel = true`) | Кнопка «Отменить» активна → BS-003 |
| Поздняя отмена (`can_cancel = false`) | Кнопка «Отменить» неактивна + пояснение «Отмена возможна не позднее чем за 6 часов» + CTA «Написать Оле» (FR-16a) |

## Матрица ошибок (LOGIC-010)
| Код | HTTP | UX-реакция |
|---|---|---|
| `slot_started` | 422 | Кнопка отмены недоступна |
| `already_cancelled` | — | Показывается текущий статус |
| Нет сети / неопределённый результат | — | Retry с тем же `Idempotency-Key` (LOGIC-008) |

---

## Навигация
- **Входящая:** SCR-006 (тап по карточке брони).
- **Исходящая:** BS-003 (тап «Отменить»); SCR-006 (успех отмены / back); Telegram (deep-link «Написать Оле»).
