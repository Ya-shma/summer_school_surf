# SCR-009 — Профиль клиента

## Цель
Данные клиента, бейджи, быстрые действия.

## Связь с требованиями
- **FR:** FR-47, FR-49, FR-59, FR-16a
- **US:** US-18, US-19, US-20
- **UC:** UC-7

## Применяемые логики
- LOGIC-007 (Паттерн состояний экрана)

---

## Структура
1. **Шапка:**
   - Имя, телефон (маскирован: `+7 *** *** ** 67`, NFR-20).
   - Бейдж «Постоянный клиент» (если `is_permanent=true`, FR-59).
   - Возраст, день рождения (если указаны).
2. **Блок «Статистика»:**
   - «Пройдено тренировок: N» (счётчик `attended`).
   - Прогресс до постоянного клиента (если ещё не): «До постоянного клиента: 10 − N».
3. **Блок «Документы»:**
   - «Правила скалодрома и ТБ» → внешний URL.
   - «Политика конфиденциальности» → внешний URL.
4. **Блок «Связь»:**
   - «Написать Оле» → deep-link `tg://` в Telegram (FR-16a).
5. **Блок «Аккаунт»:**
   - «Редактировать данные» → SCR-010.
   - «Выйти» → модалка подтверждения → очистка токенов из Keychain/Keystore (FR-49, NFR-18).

## Состояния экрана (LOGIC-007)
| Состояние | Поведение |
|---|---|
| Данные | Полная информация |
| Постоянный клиент | Бейдж «Постоянный клиент» виден |
| Не постоянный | Прогресс-бар до постоянного клиента |
| Загрузка | Skeleton |

## Навигация
- **Входящая:** tab bar.
- **Исходящая:** SCR-010 («Редактировать данные»); SCR-001 («Выйти»); Telegram (deep-link «Написать Оле»); внешние URL (документы).
